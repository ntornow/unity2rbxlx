# Phase 4a.3: Runtime Plan (Animation + Templates + Movement)

> **Last verified:** 2026-04-16. Cross-check against current `luau_validator.py` and `api_mappings.py` before acting on prescriptions.

Unity games generate content and animate at runtime. This file plans those systems from the Unity side — decisions here feed transpile (4b), storage classification (4a.5), and the spawner/animator emit in 4c.

Skip this file if the project has no `.anim`/`.controller` files AND no runtime-spawned prefabs.

## Part 1 — Animation approach

**Transform animation (legacy `.anim`, classID 111).** Auto-generated by `converter/animation_converter.py` into TweenService scripts or per-template animator config modules. Runtime side: `runtime/animator_runtime.luau`.

**Mecanim Animator on skinned meshes.** Pipeline strips skinning, searches for FBX animation files (`_Run`, `_Walk`, `_Idle`, `_Anim`), extracts Hips root motion, generates a config. Spawning code checks for animator attributes and creates an animator instance per clone. Result is a looping bob/sway — not full skeletal animation, convincing for small fast objects.

**Root motion euler sanity check.** Root motion uses **absolute bone orientations** — local euler angles in skeleton space, NOT relative offsets. Runtime animator applies euler values as **additive** rotations on top of `_baseCFrame`. Bone rotations flipping through ±180° (common in hop/tumble anims) cause wild tumbling. **Inspect the euler curve: if any axis exceeds ±45°**, replace with a hand-authored curve capturing the visual intent. Position curve is usually fine as-is.

**Particle emission classification.** For each ParticleSystem, decide **continuous** (ambient) vs **burst-triggered** (pickups, death effects). Burst particles have `rateOverTime ≈ 0` with burst entries (`m_Bursts` array, or old-format `cnt0`–`cnt3`). Pipeline sets burst emitters to `Enabled=false` with a `BurstCount` IntValue. Scripts call `emitter:Emit(burstCount)` at the right moment. Sub-emitter wiring: `runtime/sub_emitter_runtime.luau`.

**Player character animation state mapping.** For each Unity animation state (Run, Jump, Slide, Fall, Death):

1. Read `.controller` YAML to enumerate states and transition conditions.
2. Map each state to an R15 default animation or a catalog AnimationId.
3. Track with `wasX`/`isX` flags and play/stop AnimationTracks with crossfade (`Play(0.1)`/`Stop(0.1)`).
4. Priorities: base locomotion at `Action`, overlays (jump, slide) at `Action2`.

**Every Unity animation state must have a Roblox equivalent** — missing states produce T-pose snaps. Audit the controller; verify each state is covered.

**Implementability check:** a working simple version beats a broken complex one. If a system cannot be fully ported, implement an approximation and document what's missing.

## Part 2 — Templates manifest

Unity prefabs become Models in `ReplicatedStorage/Templates`. Each Model keeps its Unity prefab name. **Never auto-discover templates** by scanning name substrings or child structure — use known prefab names from the Unity project. Build a lookup table mapping names (or Unity GUIDs from data modules) to template references.

**Extract per-template metadata from Unity prefab YAML.** Read `.prefab` files to determine: template dimensions/length, sub-object spawn positions, which sub-templates can appear within a template. Hardcode this metadata in a Luau table — it cannot be derived at runtime because spatial data is lost during conversion.

**UnityLayer attributes** are set by the pipeline on scene-placed instances. Use `part:GetAttribute("UnityLayer")` for collision classification. **UnityLayer attributes are on scene instances, NOT on prefab templates.** When cloning templates at runtime, spawn code must set `UnityLayer` explicitly on cloned parts. Do not auto-discover templates by scanning for UnityLayer — use explicit name lists.

**ScriptableObject data** becomes `_Data.lua` ModuleScripts — but contains raw GUIDs that must be resolved to Template names.

## Part 3 — What spatial data does NOT survive

- **Path/spline data** (child Transforms as waypoints/curves) — stripped (`Transparency=1, CanCollide=false`). **Do not write auto-discovery code** to walk a Model's children for waypoints — it will find rendering geometry instead.
- **Normalized position values** (spawn positions as 0–1 t-values) — meaningless without the original path.
- **Collider-only geometry** (trigger volumes, invisible walls) — stripped or transparent. If gameplay depends on trigger placement within prefabs, extract that data manually.

## Part 4 — Movement model

Account for lost spatial data:

- If Unity moves objects along spline paths with lost waypoints, determine *effective* movement from the Unity code: straight-line, curved, grid-based? Port the effective movement, not the spline mechanism.
- If Unity uses `GetPointAt(t)` but the underlying path is a straight line, replace with direct position arithmetic. If genuinely curved, manually extract and hardcode control points.

**Match the pipeline's coordinate system:**

- The pipeline places objects at Unity world positions with axis flip (`(x, y, z)` → `(x, y, -z)`). Unity's forward is +Z; after flip, converted scene objects are arranged along −Z in Roblox space.
- The game loop's movement direction **must match** the axis objects are placed along. If segments lie along −Z, the character moves in −Z.
- **Character facing:** if the character should face the converted "forward", set `CFrame.Angles` accordingly — Roblox's default front face is −Z.
- **Camera placement:** use absolute world-space offsets (`characterPos + Vector3.new(0, height, behindDistance)`) rather than rotation-relative (`charCF.LookVector * distance`). Rotation-relative offsets break when character has fixed facing.

## Output

`animation_plan` and `templates_manifest` in `conversion_plan.json`:

```
animation_plan:
  transform_anims: [{anim_path, approach: "tweenservice" | "animator_config"}]
  mecanim_anims:   [{fbx_path, states: [...], root_motion_curve?}]
  particle_systems:[{path, mode: "continuous" | "burst"}]
  character_states:[{unity_state, roblox_animation_id, priority}]

templates_manifest:
  - name: "TemplatePrefabName"
    source_prefab: ".../Assets/Prefabs/X.prefab"
    metadata: { dimensions, sub_objects, ... }
    spawned_by: [list of manager class names that Instantiate this]
    unity_layer: int
    has_animator_config: bool
movement_model:
  axis: "+x" | "-x" | "+z" | "-z"
  spline_fallback: "straight" | "grid" | "curve_points"
```

`spawned_by` is the critical field for 4a.5 — it decides server vs. replicated container for the template.
